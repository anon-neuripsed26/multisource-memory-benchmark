# Cache Policy

This document describes how the frozen paper data under `data/`
relates to the LLM API calls that produced it, and what reproducibility the
shipped caches support.

---

## Two layers

### Layer 1 — Frozen paper artifacts (what's in the release)

Location:

- `data/extracted_atoms/{seed}/bench_*.json` — per-persona,
  held-out test-split LLM-extracted atoms (`μ̂`), 120 files × 4 seeds = 480 total.
- `data/method_outputs/{vendor}/{seed}/{variant}/*.json` — frozen
  per-persona LLM baseline outputs (Direct, Schema-Aware, etc.).
- `data/method_outputs/gpt-5.4/s20260321/few-shot/*.json` — frozen
  per-(persona, question) Few-Shot outputs (single seed only).
- `data/method_outputs/struct_llm/{mode}/{seed}/*.json` — frozen
  matched-input GPT condition outputs (`StructLLMSource`, mode = oracle / extracted),
  used by the 2×2 factorial decomposition figure.

These files are verbatim responses from the original batch LLM API runs that
produced the paper numbers. Filenames are **persona-keyed**, not hash-keyed.
Files contain the raw LLM text plus the parsed structured answers.

Every table in the paper is reproduced from these frozen files.
`python -m paper_artifacts.reproduce_paper` makes **zero API calls**.
For structured fusion rows reported with `\hat{\mu}` input, the reproduction
path fits/calibrates on deterministic direct-readout atoms from the train/cal
splits and evaluates on the frozen held-out test extractions above.

### Layer 2 — Live API call interface (forward-looking)

Location: `src/survey2agent/api_clients/`.

Two abstract bases — `SyncLLMClient` (single in-process completion, used by
OpenRouter) and `BatchLLMClient` (provider batch APIs, used by OpenAI and
Gemini) — define the public contract. Concrete clients (`OpenRouterSyncClient`,
`OpenAIBatchClient`, `GeminiBatchClient`) implement provider-specific transport.
This layer is provided for **downstream extension work** (new prompt, new
model, fifth seed, ablation re-run). It is **not** exercised when reproducing
paper numbers from the frozen cache.

`SyncLLMClient.complete` consults a SHA256-keyed disk cache before issuing
any network call and refuses to reach the network unless `allow_api_call=True`.
The cache record is intentionally slim. Each file contains exactly three keys:

- `cache_file_version` — schema version literal (currently `"v2"`).
- `model` — the `api_model_id` that produced the response.
- `response` — the raw response text from the provider.

`BatchLLMClient` does not consult or write the disk cache. Batch jobs are
correlated by caller-supplied `custom_id`s and surfaced as
`BatchResultItem(custom_id, text, finish_reason, error_message)`; persistence
of batch outputs is the caller's responsibility (see the frozen artifacts in
Layer 1 for the format used by the paper).

For batch providers, CLI cache-only mode is therefore a no-network guardrail:
without `--allow-api-call`, a new batch request exits instead of submitting to
the provider. It is not intended to replay the frozen `data/method_outputs/`
bundles directly; paper reproduction replays those bundles through
`paper_artifacts/`.

Unit tests under `tests/api_clients/` exercise schema validation, cache
round-trip, and batch orchestration against in-memory mock providers. They
do not touch the frozen `data/` cache.

## What the paper reports

- **Appendix Table 13** — API call **inventory** (counts per stage × vendor).
  Generated by `paper_artifacts/appendix/api_cache_inventory.py` from the
  frozen files under `data/extracted_atoms/` and `data/method_outputs/`.
- The paper does **not** report API cost figures. Cost was not tracked at
  the time of the original batch runs.

## Reproducing paper numbers

```bash
make reproduce
# equivalent to:
PYTHONPATH=src python -m paper_artifacts.reproduce_paper
```

Consumes only the frozen files under `data/`. Produces every lockable table
in the paper to within ±0.005 absolute tolerance. Zero API spend.

---

## Reproducing from scratch (optional, expensive)

If you want to regenerate the frozen artifacts from scratch:

1. Provide API keys via environment variables (`OPENAI_API_KEY`,
   `GEMINI_API_KEY`, `OPENROUTER_API_KEY`).
2. Run the extraction and baseline scripts under `scripts/` (see
   [ARCHITECTURE.md](ARCHITECTURE.md)).
3. The original runs were batch API jobs spread across 4 seeds × multiple
   vendor configurations. Re-running consumes substantial API budget. Cost
   is user-paid; this release does not ship a cost estimator.

---

## Integrity

All frozen files ship with the release tarball.
`paper_artifacts/appendix/api_cache_inventory.py` verifies per-row file-count
integrity; if any file is missing, that script reports the gap and the slow
pytest suite fails.
